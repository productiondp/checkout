# 🚀 CHECKOUT — Hyperlocal Business Matchmaking Engine

This repository contains the high-fidelity prototype and production architecture for **CHECKOUT**, an AI-powered engine that connects professionals and businesses based on intent, not browsing.

## 🏗️ Production Architecture

Built with **Next.js + Supabase + OpenAI + Razorpay**.

### 📁 Structure
- **`/src`**: Next.js App Router (Frontend Prototype)
- **`/backend`**: Business Logic & AI Services
  - `/ai`: Vector embedding generation & Hybrid matching engine
  - `/payments`: Razorpay escrow & deal settlement logic
- **`/db`**: PostgreSQL schema with `pgvector` support

### 🧠 The Relevance Engine
The core differentiator of CHECKOUT is the **Hybrid Matching Algorithm (V2)**:
```text
Score = (0.4 * Semantic Embedding) + (0.35 * Skill Overlap) + (0.15 * Location) + (0.1 * Recency/Intent)
```
- **Hard Filters**: Enforces a 30% skill threshold and location locks for physical jobs.
- **Explainability**: Every match is accompanied by a human-readable reason generated by the engine.

## ⚙️ Core Modules implemented

1. **AI Match Hub**: Personalized ranking of opportunities and talent.
2. **AI Deal Architect**: Automated budget/scope/timeline suggestions.
3. **Reputation System**: Ratings, verified badges, and reliability scores.
4. **Earnings Hub**: Secure transaction dashboard and escrow status.
5. **Growth Co-pilot**: Proactive AI assistant on the Home feed.

## 🚀 Scaling to Production

1. **Embeddings**: Replace the mock keywords in `match-engine.ts` with real OpenAI `text-embedding-3-small` calls.
2. **Database**: Apply `db/schema.sql` to a Supabase project and use `pgvector` for similarity searches.
3. **Payments**: Configure `RAZORPAY_KEY` and `RAZORPAY_SECRET` to enable live transactions.
